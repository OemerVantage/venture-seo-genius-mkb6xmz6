# Task List

## Task 1: MVP-Fokus definieren & Scope festlegen
**Category:** development
**Effort:** medium
**Deadline:** 3 days

Basierend auf dem Council Feedback, den MVP-Scope konkretisieren. Start mit WordPress-Integration (Copy-Paste Snippet & optionale REST API via n8n). Klare Abgrenzung, welche Optimierungen automatisiert werden und welche nicht. Fokus: On-page SEO, Meta-Tags, Content-Optimierungsvorschläge. KEINE direkte Code-Modifikation des Cores.

### Acceptance Criteria
- [ ] Implementation complete
- [ ] Tests written
- [ ] Documentation updated


## Task 2: User Stories & Wireframes für WordPress-MVP erstellen
**Category:** development
**Effort:** medium
**Deadline:** 5 days

Detaillierte User Stories für den MVP entwickeln, die den Workflow eines KMU-Nutzers (Login, Website verbinden, SEO-Check, Vorschläge erhalten, umsetzen mit Copy-Paste/API) abbilden. Erste Wireframes für die Kernfunktionen erstellen, um das Nutzererlebnis zu visualisieren.

### Acceptance Criteria
- [ ] Implementation complete
- [ ] Tests written
- [ ] Documentation updated


## Task 3: Validierungs-Interviews: WordPress-Agenturen & KMUs
**Category:** validation
**Effort:** high
**Deadline:** 7 days

5-10 qualitative Interviews mit potenziellen Nutzern durchführen: WordPress-Agenturen, die KMU-Seiten betreuen, und KMU-Vertreter. Fokus auf deren aktuelle SEO-Herausforderungen, die Akzeptanz von (semi-)automatisierten Lösungen und das Komfortlevel bei 'Code-Snippets' vs. 'API-Integration'. Feedback zum MVP-Scope einholen.

### Acceptance Criteria
- [ ] Implementation complete
- [ ] Tests written
- [ ] Documentation updated
